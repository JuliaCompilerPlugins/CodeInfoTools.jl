Motivation

Note

Follow along in the REPL.

Working with Core.CodeInfo is often not fun. E.g. when examining the untyped lowered form of the Rosenbrock function

CodeInfo(
1 ─ %1  = a - x
│   %2  = Core.apply_type(Base.Val, 2)
│   %3  = (%2)()
│   %4  = Base.literal_pow(Main.CodeInfoToolsTour.:^, %1, %3)
│   %5  = Core.apply_type(Base.Val, 2)
│   %6  = (%5)()
│   %7  = Base.literal_pow(Main.CodeInfoToolsTour.:^, x, %6)
│   %8  = y - %7
│   %9  = Core.apply_type(Base.Val, 2)
│   %10 = (%9)()
│   %11 = Base.literal_pow(Main.CodeInfoToolsTour.:^, %8, %10)
│   %12 = b * %11
│   %13 = %4 + %12
└──       return %13
)

Do you ever wonder – is there another (perhaps, any) way to work with this object? A Builder perhaps? Where I might load my CodeInfo into – iterate, make local changes, and produce a new copy?

Contribution

CodeInfoTools.jl provides a Builder abstraction (amongst a few other nifty tools) which allows you to safely iterate over and manipulate Core.CodeInfo. It also provides more advanced functionality for creating and evaluating Core.CodeInfo – which is a bit on the experimental side.

How might you use this in practice?

using CodeInfoTools

rosenbrock(a, b, x, y) = (a - x)^2 + b * (y - x^2)^2

display(code_inferred(rosenbrock, Int, Int, Int, Int))

function swap(st)
    st isa Expr || return st
    st.head == :call || return st
    st.args[1] == Base.:(+) || return st
    return Expr(:call, Base.:(*), st.args[2:end]...)
end

function transform(src)
    b = CodeInfoTools.Builder(src)
    for (v, st) in b
        b[v] = swap(st)
    end
    return finish(b)
end

src = code_info(rosenbrock, Int, Int, Int, Int)
new = transform(src)

display(src)
display(new)

Here, we've lowered a function directly to a Core.CodeInfo instance and created a Builder instance b. You can now safely iterate over this object, perform local changes, press finish and - (la di da!) - out comes a new Core.CodeInfo with your changes fresh.
